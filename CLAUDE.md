# Anita Comics

## Overview
Anita Comics is a lightweight, xkcd-inspired comic strip viewer hosted on GitHub Pages. Comics are loaded from a `manifest.json` file that is auto-generated by a GitHub Action whenever a new photo is pushed — no code editing needed to publish new strips.

## Project Structure
```
Anita_comics/
├── index.html                        # Page structure
├── style.css                         # Styling (light-blue theme, xkcd-inspired)
├── code.js                           # Reads manifest.json + navigation logic
├── manifest.json                     # Auto-generated list of comic filenames
├── start.command                     # Double-click to run locally (Mac)
├── CLAUDE.md                         # This file
├── photos/                           # Drop comic images here
│   └── January_5_2025.jpeg           # ← naming format
└── .github/
    └── workflows/
        └── update-manifest.yml       # Auto-updates manifest.json on push
```

## How to Publish on GitHub Pages

### First-time setup
1. Push this folder to a GitHub repository
2. Go to **Settings → Pages** in the repo
3. Under *Branch*, select `main` and click **Save**
4. Your site will be live at `https://your-username.github.io/your-repo-name/`

### Adding new comics (ongoing)
1. Name the image file: `Month_Day_Year.jpeg`  
   Example: `March_22_2026.jpeg`
2. Upload it to the `photos/` folder on GitHub (drag & drop on the GitHub website)
3. The GitHub Action runs automatically within ~30 seconds and updates `manifest.json`
4. Refresh the site — the new comic is live!

> **Month must be spelled in full:** January, February, March, April, May, June, July, August, September, October, November, December

## Running Locally
Double-click **start.command**. It will:
1. Scan `photos/` and regenerate `manifest.json`
2. Start a local server
3. Open `http://localhost:8000` in your browser

Close the Terminal window when done.

## How manifest.json Works
- `manifest.json` is just a list of comic filenames: `["January_1_2025.jpeg", ...]`
- The GitHub Action (`.github/workflows/update-manifest.yml`) regenerates it automatically on every push that touches `photos/`
- `code.js` fetches `manifest.json` on page load, parses dates from filenames, sorts them oldest-first, and shows the most recent comic

## Navigation
- **← Prev** — one comic back; disabled on the first comic
- **Next →** — one comic forward; disabled on the last comic
- **Random** — jump to a random comic

## Changing the Header Image
Open `code.js` and update the `title` variable at the top:
```js
const title = "your-new-image-url-here";
```
